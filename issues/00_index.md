# Issue Index: Message-First Communication Compiler

## Purpose

This issue set defines the implementation plan for evolving the existing `minimalist-presentation-mcp` repository into a contest-ready application for the Zenn Microsoft Agent Hackathon.

The product must not be presented as a generic slide generator. The target concept is:

> A Communication Compiler that converts a user's rough thoughts into a sharpened business message, validates the message through evidence and objections, and then expands it into a five-page Message-First Deck.

The final slide deck is the UI. The real value is the process that extracts, diagnoses, and refines what the user actually wants to say.

## Target Architecture

```text
User
  ↓
Simple Web UI
  - Free talk input
  - Claim candidates
  - 4-Line Message Kernel
  - Diagnosis / evidence gaps / objections
  - Idea Map
  - Deck / Brief output links
  ↓
App Backend
  - Session state
  - Agent run orchestration
  - Output persistence
  ↓
Azure AI Foundry Agent
  - Understands rough user input
  - Extracts claim candidates
  - Builds/refines the 4-Line Message Kernel
  - Diagnoses contradictions and evidence gaps
  - Calls tools
  ↓
Tools
  ├─ Existing Minimalist Presentation MCP
  │    - get_presentation_guideline
  │    - create_message_first_deck
  ├─ Evidence Search / internal data mock
  ├─ Brief Generator
  └─ Optional Image Generator
```

## Core Product Terms

### Communication Compiler

The overall product concept. It compiles rough user intent into a structured business communication artifact.

### Free Talk

A raw input from the user. It may be a transcript, memo, rambling text, meeting note, or unstructured idea.

### Claim Candidate

One of several possible interpretations of what the user really wants to say.

### 4-Line Message Kernel

The intermediate representation before deck generation.

```text
1. Audience Premise
2. Complication / Gap
3. Core Claim
4. Requested Decision / Action
```

The Agent must refine this kernel before generating the deck.

### Message-First Deck

A fixed five-page deck generated by the existing MCP server.

```text
M1: Core Thesis
M2: Constraint / Strategy
M3: Decision / Action
E1: Evidence 1
E2: Evidence 2
```

## P0 Scope

The minimum contest-ready version must include:

- Simple Web UI
- Free Talk input
- Azure AI Foundry Agent orchestration
- Three claim candidates
- 4-Line Message Kernel
- Kernel diagnosis
- Evidence gap / objection extraction
- MCP deck generation
- Generated deck URL display
- Demo scenario and documentation

## P1 Scope

Add if time permits:

- Distributable Markdown / HTML brief
- Idea Map visualization
- Kernel revision history
- Evidence source cards
- Agent trace summary

## P2 Scope

Add only after P0/P1 is stable:

- GPT Image 2 visual concept generation
- PowerPoint export
- Teams / SharePoint real integration
- Voice input
- Multi-audience rewriting

## Global Non-Goals

Do not implement the following in the first contest version:

- Full enterprise authentication
- Full SharePoint crawling
- Advanced vector search infrastructure unless already available
- Complex multi-agent debate
- Full PowerPoint export
- A production-grade document management system

## Required Product Narrative

All implementation and documentation must reinforce the following narrative:

> The application does not simply create slides. It helps users discover and sharpen what they actually want to say, compresses it into a 4-Line Message Kernel, diagnoses weaknesses, grounds the claim with evidence and objections, and then expands the refined message into a five-page Message-First Deck.
